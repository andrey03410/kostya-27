# Технологический стек

> Статус: **финализировано** (2026-06-02). См. `decisions.md`.
>
> **Выбрано:** Vite + React · Framer Motion · **Tailwind CSS** · Canvas/tsParticles ·
> Howler.js · Zustand. Формат — **скролл-сторителлинг** (без react-router).

## База (уже есть)
- **Vite** — сборка и dev-сервер.
- **React** — UI.

## Предлагается добавить

| Область | Кандидат | Зачем | Альтернатива |
|---|---|---|---|
| Анимации | **Framer Motion** | Декларативные анимации, переходы, жесты, layout-анимации | GSAP |
| Частицы | **tsParticles** или собственный **Canvas** | Молнии, сакура, данго | react-particles |
| Звук | **Howler.js** | Кросс-браузерное аудио, управление громкостью | нативный Audio API |
| Состояние | **Zustand** | Лёгкое глобальное состояние (счёт, прогресс) | React Context |
| Навигация | **react-router** *(если многостраничный вариант)* | Маршруты между экранами | скролл-секции без роутера |
| Стили | **CSS-переменные + CSS Modules** | Контроль и простота | Tailwind CSS |
| Конфетти/салют | **canvas-confetti** | Готовый красивый салют | свой Canvas |

## Развилки для обсуждения
1. **Скролл-сторителлинг** (одна страница, секции, анимации по скроллу)
   **vs** **роутинг** (отдельные экраны: интро → хаб → игры → финал).
2. **Tailwind** vs **CSS Modules** — скорость стилизации против минимума зависимостей.
3. **Zustand** vs **Context** — зависит от того, насколько сложным будет состояние игр.
